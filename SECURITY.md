# Security Policy

## Status

This repository is a quiet beta for local Claude Code configuration and
verification. It is not a hosted service, but security still matters.

## Report Vulnerabilities Privately

Do not open a public issue with exploit details.

Use one of these paths:

1. GitHub private vulnerability reporting if available
2. a private maintainer contact path
3. a minimal issue requesting secure contact, without technical details

## Useful Report Contents

- affected script or workflow
- impact summary
- reproduction steps
- whether tokens, local files, or config exposure is involved

## Expectations

- support bundles should avoid leaking secrets by default
- version and config checks should not expose unrelated private data
- fixes should be coordinated privately before public disclosure
