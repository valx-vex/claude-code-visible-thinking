# Quiet Beta Operations

## Who This Beta Is For

This beta is for users who:

- already use Claude Code
- want visible thinking summaries or transcript-visible reasoning back
- can follow terminal instructions
- prefer a reproducible setup over folklore

## What Feedback Is Most Valuable

- version-specific breakage in recent Claude Code stable builds
- verification failures with clear machine state
- documentation gaps that confuse first-time users
- places where the support bundle is missing the detail needed to debug fast

## Supported Version Contract

This repo only treats recent stable Claude Code builds as supported.

If a node is below the documented supported floor, the verifier should say so
explicitly instead of producing a vague failure.

## GitHub Discussion Mapping

Use GitHub Discussions as the broad feedback layer:

- `Announcements` for maintainer updates
- `General` for beta feedback
- `Q&A` for install help and usage questions
- `Show and tell` for successful setups
- `Ideas` for roadmap discussion

## Discord Shape

The recommended starter Discord layout is:

- `#announcements`
- `#start-here`
- `#install-help`
- `#bug-triage`
- `#show-your-setup`
- `#contributors`

Discord is for conversation. GitHub is where the project remembers.
