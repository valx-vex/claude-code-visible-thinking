# Optional Memory Stack

This repository is about visible thinking behavior in Claude Code.

Persistent memory is a separate optional add-on and should stay separate in
messaging and implementation:

- visible thinking: UI / transcript behavior
- MemPalace: verbatim local memory
- Lazarus: semantic resurrection
- VexNet continuity: capture and sync substrate

## Practical implication

If you document memory here, keep it explicitly optional and point to the
canonical fleet integration assets rather than embedding unrelated private
architecture into the visible-thinking setup.
